

## Plan: Use PDF branding images (header/footer) in document HTML fallback views

### Problem
When the client clicks "Ver" on the signature portal (`/firmar/:token`) or "Descargar" in the Documentos tab, if no signed/base PDF exists yet, the system falls back to rendering the HTML `content` in a print window. Currently:
- **SignatureView.tsx**: Uses the generic `logo_url` from companies table, not the dedicated `pdf_header_image_url` / `pdf_footer_image_url` from `company_settings`
- **DocumentsManager.tsx**: Shows no branding at all in the HTML fallback

The actual signed PDFs (generated by `generate-base-pdf` via Puppeteer) correctly use the SAMAP header/footer images, but the HTML preview fallback does not match.

### Solution

**1. Fetch PDF branding images in `useSignatureLinkByToken`** (`src/hooks/useSignatureLinkPublic.ts`)
- After fetching sale data, also query `company_settings_public` (the public view) to get `pdf_header_image_url` and `pdf_footer_image_url` for the sale's company
- Include these URLs in the returned `linkData` object

**2. Update HTML fallback in `SignatureView.tsx`** (lines 137-189)
- Replace the generic `logo_url` header with an `<img>` tag using `pdf_header_image_url` (full-width banner)
- Replace the text-based footer with an `<img>` tag using `pdf_footer_image_url`
- Keep the existing CSS fixed positioning but adjust dimensions to match the Puppeteer output (header: 24mm, footer: 16mm)

**3. Update HTML fallback in `DocumentsManager.tsx`** (lines 127-146)
- Fetch `pdf_header_image_url` and `pdf_footer_image_url` from `company_settings` (authenticated query)
- Add fixed-position header/footer images to `openHtmlContentWindow`, matching the same layout as SignatureView

### Technical Details
- The `company_settings_public` view is accessible via the signature token (RLS already configured)
- If the view doesn't expose `pdf_header_image_url`/`pdf_footer_image_url`, we'll query `company_settings` directly using the signature client (which has access via `get_sale_id_from_signature_token`)
- For `DocumentsManager` (authenticated context), we query `company_settings` using the standard supabase client with the user's profile `company_id`

### Files Changed
- `src/hooks/useSignatureLinkPublic.ts` — add company_settings branding fetch
- `src/pages/SignatureView.tsx` — use branding images in HTML fallback
- `src/components/DocumentsManager.tsx` — add branding images to HTML fallback
